# F02 — 字圖搜尋

> 功能類別：核心搜尋功能
> 相關檔案：`lib/glyphs.ts`、`app/api/glyphs/route.ts`、`app/page.tsx`

---

## 1. 功能概述

字圖搜尋是 MoScript 的核心功能。使用者輸入中文文字（可多字），系統依書體、結果範圍、排序方式等條件，查詢並回傳每個字的多筆書法字圖候選結果。前端支援無限捲動分頁，每頁最多回傳 20 個字。

---

## 2. 搜尋條件規格

### 2.1 搜尋文字（q）

- 僅接受 Unicode CJK 中文字符
- 非中文字元自動過濾（包含數字、英文、標點）
- 多字查詢：輸入多個字，每字分別查詢
- 空白查詢：可瀏覽書庫（不限定字）

### 2.2 書體篩選（scriptTypes）

| 書體值 | 中文名稱 |
|--------|---------|
| `草` | 草書 |
| `行` | 行書 |
| `隸` | 隸書 |
| `楷` | 楷書 |
| `篆` | 篆書 |
| `魏` | 魏碑 |
| `其他` | 其他書體 |

- 可多選（陣列）
- 空陣列代表不限書體

### 2.3 結果範圍（resultScope）

| 值 | 說明 | 登入需求 |
|-----|------|---------|
| `library` | 僅系統書庫字圖（owner_user_id IS NULL） | 不需要 |
| `all` | 公開字圖＋個人字圖（需登入） | 需要 |
| `liked` | 使用者已按讚的字圖 | 需要 |
| `personal` | 使用者本人上傳的字圖 | 需要 |
| `public` | 所有公開字圖 | 不需要 |

### 2.4 排序方式（sort）

| 值 | 排序邏輯 | 說明 |
|-----|---------|------|
| `popular` | `(like_count + collection_count × 10) DESC`，次排 quality_score，再排 id DESC | 熱門度優先 |
| `newest` | `id DESC` | 最新上傳優先 |
| `author` | 有作者者優先（未知排後），作者字母順序 | 依書家姓名 |
| `script` | 書體優先級（草>行>隸>楷>篆>魏>其他），次排 quality_score | 依書體分類 |

### 2.5 每字結果數（perChar）

- 預設：3 筆
- 最大值：10 筆
- 超過上限自動截斷

### 2.6 分頁（offset）

- 以字為單位分頁
- 每頁最多 20 個字
- 前端以無限捲動（Intersection Observer）自動載入下一頁

## 3. API 規格

### GET /api/glyphs

**請求參數（Query String）：**

| 參數名稱 | 類型 | 必填 | 預設值 | 說明 |
|---------|------|------|--------|------|
| `q` | string | 否 | `""` | 搜尋文字（中文） |
| `scriptTypes` | string（逗號分隔）| 否 | `""` | 書體篩選 |
| `resultScope` | string | 否 | `library` | 結果範圍 |
| `sort` | string | 否 | `popular` | 排序方式 |
| `perChar` | number | 否 | `3` | 每字結果數 |
| `offset` | number | 否 | `0` | 分頁偏移量（字數） |
| `limit` | number | 否 | `20` | 每頁字數 |

**成功回應（200 OK）：**
```json
{
  "glyphs": [
    {
      "char": "山",
      "results": [
        {
          "id": 1,
          "char": "山",

          "source": "manual",
```

**錯誤回應：**

| HTTP 狀態碼 | 情境 |
|------------|------|
| 400 | 非法的 perChar 值（<1 或 >10） |
| 401 | resultScope 需要登入但未登入 |

## 4. 查詢邏輯（SQL）

系統使用 SQLite WITH 子句與視窗函數（`ROW_NUMBER() OVER (PARTITION BY char)`）實現每字限量查詢：

```sql
WITH ranked AS (
  SELECT
    g.*,
    COALESCE(lc.like_count, 0) AS like_count,
    COALESCE(cc.collection_count, 0) AS collection_count,
    COALESCE(ll.liked_by_me, 0) AS liked_by_me,
    ROW_NUMBER() OVER (
      PARTITION BY g.char
      ORDER BY [排序條件]
    ) AS rn
  FROM glyphs g
  LEFT JOIN (SELECT glyph_id, COUNT(*) AS like_count FROM glyph_likes GROUP BY glyph_id) lc ON g.id = lc.glyph_id
  LEFT JOIN (SELECT glyph_id, COUNT(*) AS collection_count FROM collection_items GROUP BY glyph_id) cc ON g.id = cc.glyph_id
  LEFT JOIN (SELECT glyph_id, 1 AS liked_by_me FROM glyph_likes WHERE user_id = ?) ll ON g.id = ll.glyph_id
  WHERE [篩選條件]
)
SELECT * FROM ranked
WHERE rn <= :perChar
ORDER BY [字順序], rn
LIMIT :limit OFFSET :offset
```

## 5. 前端行為

### 5.1 中文輸入法處理

- 監聽 `compositionstart` / `compositionend` 事件
- 輸入法組字期間（composition 中）不觸發搜尋
- 組字完成後才發送 API 請求

### 5.2 即時搜尋

- 搜尋條件變更後立即發送請求（無 debounce 延遲）
- 書體、範圍、排序等篩選條件變更立即重新搜尋

### 5.3 無限捲動

- 使用 `Intersection Observer` 監測底部元素
- 滾動至底部時自動載入下一頁
- `hasMore: false` 時停止載入

### 5.4 集字選擇

- 搜尋結果以字為組顯示（每字多個選項）
- 點擊字圖選取（同字只能選一個）
- 選取後高亮顯示，底部工具列出現「儲存集字」按鈕

## 6. 效能設計

| 機制 | 說明 |
|------|------|
| 複合索引 | `idx_glyphs_char_script` 加速按字＋書體查詢 |
| 視窗函數分組 | 一次 SQL 即可取得每字 TOP N 結果，不需 N+1 查詢 |
| 縮圖支援 | 列表顯示使用 `thumbnail_url`，詳情頁才載入完整圖 |
| 分頁 | 每頁 20 字，避免一次載入過多資料 |
