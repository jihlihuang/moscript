# F05 — 個人字圖管理

> 功能類別：使用者內容管理
> 相關檔案：`app/api/me/glyphs/`、`app/me/`、`app/upload/edit/[id]/`、`components/PersonalGlyphManager.tsx`

---

## 1. 功能概述

登入使用者可在「個人頁面」（`/me`）管理自己上傳的書法字圖，包含查看清單、修改元資料、切換公開/私密、以及刪除字圖。支援批量操作（一次刪除多張、一次改變可見性）。

---

## 2. 可執行操作

| 操作 | 說明 | 批量 |
|------|------|------|
| 查看清單 | 列出自己上傳的所有字圖 | — |
| 編輯元資料 | 修改書體、作者、作品名稱 | 否 |
| 切換可見性 | 在 public / private 之間切換 | 是 |
| 刪除字圖 | 刪除字圖及實體檔案 | 是 |

---

## 3. 功能流程

### 3.1 查看個人字圖清單

```
GET /api/me/glyphs
  │
  ├── 驗證 Session（需登入）
  ├── 查詢 glyphs WHERE owner_user_id = 當前使用者 ID
  └── 回傳字圖清單（含 like_count, collection_count）
```

### 3.2 編輯單一字圖

```
使用者至 /upload/edit/[id]
  │
  ├── 載入字圖資料（GET /api/glyphs/[id]）
  ├── 填寫可編輯欄位（書體、作者、作品名稱）
  └── 提交 PATCH /api/me/glyphs（或 PATCH /api/glyphs/[id]）
        ├── 驗證為字圖所有者
        ├── 更新 glyphs 表中的 script_type, author, work_title
        └── 同步資料庫至 Blob
```

### 3.3 批量操作流程

```
使用者在 /me 頁面勾選字圖
  │
  ├── 批量刪除：
  │     └── PATCH /api/me/glyphs { action: 'delete', ids: [1,2,3] }
  │           ├── 驗證每個 id 都屬於當前使用者
  │           ├── 刪除 glyphs 記錄（CASCADE 刪除相關 likes, collection_items）
  │           ├── 刪除本機實體檔案（或 Blob）
  │           └── 同步資料庫
  │
  └── 批量切換可見性：
        └── PATCH /api/me/glyphs { action: 'visibility', ids: [1,2,3], visibility: 'public' }
              ├── 驗證每個 id 都屬於當前使用者
              ├── 更新 visibility 欄位
              ├── 移動實體檔案（public ↔ private 目錄）
              └── 更新 image_url（路徑改變）
```

## 4. API 規格

### 4.1 GET /api/me/glyphs
**用途**：列出個人上傳的字圖

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入 |

**成功回應（200 OK）：**
```json
[
  {
    "id": 1,
    "char": "山",
    "author": "王羲之",
    "script_type": "行",
    "work_title": "蘭亭集序",
    "image_url": "/glyphs/山/王羲之_行_蘭亭集序_001.jpg",
    "thumbnail_url": null,
    "visibility": "public",
    "like_count": 5,
    "collection_count": 2,
    "created_at": "2025-01-01T00:00:00Z"
  }
]
```

### 4.2 PATCH /api/me/glyphs
**用途**：批量操作個人字圖

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入 |

**請求 Body（批量刪除）：**
```json
{
  "action": "delete",
  "ids": [1, 2, 3]
}
```

**請求 Body（批量切換可見性）：**
```json
{
  "action": "visibility",
  "ids": [1, 2, 3],
  "visibility": "private"
}
```

**請求 Body（更新元資料）：**
```json
{
  "action": "update",
  "id": 1,
  "scriptType": "草",
  "author": "新作者",
  "workTitle": "新作品名"
}
```

**成功回應：**
- 刪除：`{ "deleted": 3 }`
- 切換可見性：`{ "updated": 3 }`
- 更新元資料：`{ "updated": true }`

**錯誤回應：**

| HTTP 狀態碼 | 情境 |
|------------|------|
| 401 | 未登入 |
| 403 | 嘗試操作不屬於自己的字圖 |
| 404 | 字圖 ID 不存在 |
| 400 | action 欄位無效 |

### 4.3 DELETE /api/me/glyphs/[id]
**用途**：刪除單一個人字圖

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入（且為字圖所有者） |
| 成功回應 | 204 No Content |
| 副作用 | 刪除本機或 Blob 上的實體檔案 |

---

## 5. 可編輯欄位

| 欄位 | 可否修改 | 說明 |
|------|---------|------|
| char（中文字） | 否 | 建立後不可更改 |
| author（書寫者） | 是 | |
| script_type（書體） | 是 | |
| work_title（作品名稱） | 是 | |
| visibility（可見性） | 是 | 切換公開/私密 |
| image_url（圖片） | 否 | 建立後不可更換圖片 |

---

## 6. 可見性切換的檔案遷移

當字圖從 `private` 改為 `public`（或反向）時：

| 動作 | 來源路徑 | 目標路徑 |
|------|---------|---------|
| private → public | `data/private-glyphs/[char]/[file]` | `public/glyphs/[char]/[file]` |
| public → private | `public/glyphs/[char]/[file]` | `data/private-glyphs/[char]/[file]` |

同時更新 `glyphs.image_url` 與 `glyphs.thumbnail_url` 欄位的路徑。

Azure Blob Storage 模式下，則移動 Blob 路徑（`glyphs/` ↔ `private-glyphs/`）。

---

## 8. 業務規則

1. **所有者驗證**：所有操作均需驗證 `owner_user_id = 當前使用者`
2. **管理員例外**：管理員可操作任何字圖（透過後台 API）
3. **刪除連鎖效應**：刪除字圖後，相關的 `glyph_likes` 與 `collection_items` 也會串聯刪除
4. **集字作品影響**：字圖刪除後，引用它的集字作品項目也會消失（使用者需注意）
