# F04 — 字圖上傳

> 功能類別：內容貢獻
> 相關檔案：`lib/glyph-upload.ts`、`lib/blob-storage.ts`、`app/api/glyphs/upload/`、`app/api/admin/upload/`、`app/api/glyph-sets/`、`app/upload/`、`components/AdminGlyphUploadForm.tsx`

---

## 1. 功能概述

登入使用者（及管理員）可上傳書法字圖到系統，支援兩種上傳模式：

| 模式 | 說明 | API 端點 |
|------|------|---------|
| 單張上傳 | 一次上傳一個字圖，手動填寫元資料 | `POST /api/glyphs/upload`（一般使用者）或 `POST /api/admin/upload`（管理員） |
| 多字拆圖批次上傳 | 上傳一張含多個字的原稿圖，系統自動拆分後逐一上傳 | 同上，配合 `POST /api/glyph-sets` 建立字組 |

管理員於後台（`/admin/upload`）上傳，一般使用者於個人頁（`/upload`）上傳。

---

## 2. 上傳限制

| 限制項目 | 值 |
|---------|-----|
| 最大檔案大小 | 20 MB |
| 速率限制（一般使用者） | 10 次/分鐘/IP |
| 支援格式 | jpg、jpeg、png、webp、gif、bmp、ico、avif、heic、heif、tif、tiff |
| 必填欄位 | 中文字（char）、圖片檔案、**書體（scriptType）** |

---

## 3. 功能流程

### 3.1 單張上傳流程

```
使用者至上傳頁面
  │
  ├── 填寫表單：
  │     ├── 中文字（必填，僅接受單一 CJK 字元）
  │     ├── 書體（必填）
  │     ├── 作者（必填）
  │     ├── 作品名稱（必填）
  │     ├── 來源（必填）
  │     ├── 授權（必填）
  │     └── 可見性（public / private）
  │
  ├── 選擇圖片、前端處理（黑白化/擦除）
  │
  └── 提交 API
        ├── 驗證身份與欄位
        ├── 儲存圖片（本機或 Azure Blob）
        ├── 寫入 glyphs 資料表
        ├── 記錄 usage_event
        └── 同步資料庫至 Blob
```

### 3.2 多字拆圖批次上傳流程

```
使用者上傳原稿圖（含多字）
  │
  ├── 前端拆字（自動/手動框選）
  │     ├── 拆字方向：自動、橫排、直排、手動框選
  │     └── 可逐一調整每字圖像（裁切、擦除、顏色模式）
  │
  ├── 填入每字的中文字（支援快速填入整段文字）
  │
  ├── 點擊「批次上傳」
  │     ├── 驗證共用元資料（書體必填）
  │     │
  │     ├── 建立字組 POST /api/glyph-sets
  │     │     ├── 上傳原稿圖
  │     │     └── 名稱 = 批次字元合集（如「小橋流水人家」）
  │     │
  │     └── 逐一上傳每個字圖（含 setId + setPosition）
  │           └── 顯示進度條（已完成數 / 總數）
  │
  └── 全部成功後清除上傳檔案及拆字結果（字圖元資料欄位保留）
```

## 4. API 規格

### POST /api/glyphs/upload（一般使用者）

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入 |
| Content-Type | `multipart/form-data` |
| 速率限制 | 10 次/分鐘（IP 級） |

**表單欄位：**

| 欄位名稱 | 類型 | 必填 | 說明 |
|---------|------|------|------|
| `file` | File | 是 | 圖片檔案（最大 20 MB） |
| `thumbnailFile` | File | 否 | 縮圖檔案 |
| `char` | string | 是 | 中文字（單一 CJK 字元） |
| `scriptType` | string | 是 | 書體（前端驗證必填） |
| `author` | string | 是 | 書寫者名稱 |
| `workTitle` | string | 是 | 作品名稱 |
| `source` | string | 是 | 來源 |
| `license` | string | 是 | 授權類型 |
| `visibility` | string | 否 | `public` 或 `private`（預設 `public`） |
| `setId` | string | 否 | 所屬字組 ID（批次上傳時填入） |
| `setPosition` | string | 否 | 在字組中的位置（1-based，批次上傳時填入） |

### POST /api/admin/upload（管理員）

與 `/api/glyphs/upload` 欄位相同，差異：

- 不需要速率限制
- 字圖寫入系統書庫（`owner_user_id = NULL`）
- 支援 `qualityScore` 欄位

### POST /api/glyph-sets（建立字組）

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入 |
| Content-Type | `multipart/form-data` |

**表單欄位：**

| 欄位名稱 | 類型 | 必填 | 說明 |
|---------|------|------|------|
| `sourceImage` | File | 否 | 拆字原稿圖 |
| `name` | string | 否 | 字組名稱（預設為批次字元合集） |
| `visibility` | string | 否 | 可見性 |

## 5. 儲存路徑邏輯

| 情境 | 路徑 |
|------|------|
| 公開字圖（本機） | `public/glyphs/[char]/[filename]` |
| 私密字圖（本機） | `data/private-glyphs/[char]/[filename]` |
| 公開字圖（Azure） | Blob：`glyphs/[char]/[filename]` |
| 私密字圖（Azure） | Blob：`private-glyphs/[char]/[filename]` |
