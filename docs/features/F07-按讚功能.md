# F07 — 按讚功能

> 功能類別：使用者互動
> 相關檔案：`lib/glyph-stats.ts`、`app/api/glyphs/[id]/like/`、`components/GlyphLikeButton.tsx`

---

## 1. 功能概述

登入使用者可對字圖按讚（Like），表達對某件書法字圖的喜好。按讚數也是搜尋排序的重要因子（`popular` 排序中，按讚數直接影響排名）。同一使用者對同一字圖只能按讚一次，再次點擊則取消按讚。

---

## 2. 功能規格

| 項目 | 規格 |
|------|------|
| 按讚對象 | 字圖（glyphs） |
| 登入需求 | 必須登入才能按讚 |
| 速率限制 | 30 次/分鐘（使用者級） |
| 重複按讚 | 切換（按讚 ↔ 取消按讚） |
| 唯一性 | 複合主鍵 (glyph_id, user_id)，同一使用者同一字圖只能一筆 |

---

## 3. 功能流程

### 3.1 按讚流程

```
使用者點擊按讚按鈕
  │
  ├── 前端立即樂觀更新（Optimistic Update）：
  │     └── 按讚數 +1，按鈕狀態改為「已按讚」
  │
  ├── 發送 POST /api/glyphs/[id]/like
  │     ├── 速率限制檢查（30 次/分鐘/使用者）
  │     ├── Session 驗證（需登入）
  │     ├── 查詢是否已按讚
  │     │     ├── 未按讚 → INSERT INTO glyph_likes
  │     │     └── 已按讚 → DELETE FROM glyph_likes（取消按讚）
  │     └── 同步資料庫至 Blob
  │
  └── 前端依回應更新最終狀態
        └── 失敗時回滾樂觀更新
```

---

## 4. API 規格

### POST /api/glyphs/[id]/like
**用途**：切換字圖按讚狀態

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入 |
| 速率限制 | 30 次/分鐘（使用者 ID 為 key） |

**路由參數：**

| 參數 | 說明 |
|------|------|
| `id` | 字圖 ID |

**成功回應（200 OK）：**
```json
{
  "liked": true,
  "like_count": 13
}
```
- `liked: true` — 操作結果為按讚
- `liked: false` — 操作結果為取消按讚

**錯誤回應：**

| HTTP 狀態碼 | 情境 |
|------------|------|
| 401 | 未登入 |
| 404 | 字圖不存在 |
| 403 | 字圖無存取權限（私密字圖） |
| 429 | 超過速率限制（30 次/分鐘） |

---

## 5. 按讚對搜尋排序的影響

按讚數用於 `popular` 排序計算：

```sql
-- 熱門度分數公式
score = like_count + collection_count * 10
```

字圖被越多人按讚，`popular` 排序中排名越前。集字數的權重（×10）高於按讚數，代表系統更重視實際被使用的字圖。

---

## 6. 前端組件

| 組件 | 用途 |
|------|------|
| `GlyphLikeButton.tsx` | 按讚按鈕（含計數顯示、樂觀更新） |

**按鈕狀態：**
| 狀態 | 顯示 |
|------|------|
| 未登入 | 灰色按鈕（點擊提示需登入） |
| 已登入/未按讚 | 空心愛心圖示 + 按讚數 |
| 已登入/已按讚 | 實心紅色愛心 + 按讚數 |
| 載入中 | 禁用按鈕 |

---

## 7. 資料一致性

- 字圖刪除時，相關的 `glyph_likes` 記錄串聯刪除（`ON DELETE CASCADE`）
- 搜尋 API 即時計算 `like_count`（`LEFT JOIN` subquery），不快取
- `liked_by_me` 欄位依當前使用者 ID 即時查詢
