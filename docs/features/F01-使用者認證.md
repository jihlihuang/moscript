# F01 — 使用者認證

> 功能類別：身份驗證與授權
> 相關檔案：`lib/auth.ts`、`app/api/auth/`、`middleware.ts`

---

## 1. 功能概述

系統使用 Google OAuth 2.0 作為唯一登入方式。使用者透過 Google 帳號授權後，系統以 AES-256-GCM 加密的 Cookie 維持登入狀態（Session），無需另外維護密碼資料庫。管理員權限透過環境變數白名單控制，不需要額外的角色資料表。

---

## 2. 使用者角色

| 角色 | 說明 | 判斷方式 |
|------|------|---------|
| 訪客 | 未登入的一般使用者 | 無 Session Cookie |
| 一般使用者 | 已登入的 Google 帳號持有者 | Session Cookie 有效 |
| 管理員 | 具有後台操作權限的使用者 | email 在 `MOSCRIPT_ADMIN_EMAILS` 環境變數清單中 |

---

## 3. 功能流程

### 3.1 Google OAuth 登入流程

```
使用者                       系統                        Google
  │                           │                              │
  │── 點擊「Google 登入」 ──▶│                              │
  │                           │── 產生 state (CSRF 防護) ──│
  │                           │── 產生 code_verifier (PKCE)│
  │                           │── 儲存至 httpOnly Cookie ──│
  │◀── 重導向至 Google 授權頁 ─│                              │
  │                           │                              │
  │──────────────── 使用者授權 ────────────────────────────▶│
  │◀──────────────── 重導向回 /api/auth/google/callback ────│
  │                           │                              │
  │                           │── 驗證 state 參數 ─────────│
  │                           │── 交換 authorization_code ─▶│
  │                           │◀─ 取得 access_token ────────│
  │                           │── 取得使用者資訊 ───────────▶│
  │                           │◀─ name, email, picture ─────│
  │                           │                              │
  │                           │── Upsert users 資料表 ───────│
  │                           │── 加密 Session Cookie ───────│
  │◀── 重導向至首頁，設定 Cookie ─│                          │
```

### 3.2 Session 驗證流程

```
每次 API 請求
  │
  ├── 讀取 session Cookie
  ├── Base64URL 解碼
  ├── 拆解：iv[12 bytes] | ciphertext | authTag[16 bytes]
  ├── AES-256-GCM 解密 (金鑰來自 AUTH_SECRET)
  ├── JSON.parse 取得 { userId, email, name, picture, isAdmin }
  └── 回傳使用者物件（驗證失敗則回傳 null）
```

### 3.3 登出流程

```
使用者點擊登出
  │
  ├── POST /api/auth/logout
  ├── 驗證 CSRF (需已登入)
  ├── 清除 session Cookie (設定為空值、過期)
  └── 重導向至首頁
```

## 4. API 規格

### 4.1 GET /api/auth/google
**用途**：啟動 Google OAuth 授權流程

| 項目 | 內容 |
|------|------|
| 認證需求 | 不需要 |
| 請求參數 | 無 |
| 成功回應 | 302 重導向至 Google 授權頁 |
| 副作用 | 設定 `oauth_state` 與 `pkce_verifier` httpOnly Cookie（5 分鐘過期） |

### 4.2 GET /api/auth/google/callback
**用途**：處理 Google OAuth 回調

| 項目 | 內容 |
|------|------|
| 認證需求 | 不需要 |
| Query 參數 | `code`（授權碼）、`state`（CSRF 驗證碼）、`error`（可選，錯誤訊息） |
| 成功回應 | 302 重導向至首頁，設定加密 Session Cookie（14 天過期） |
| 失敗情境 | state 不符 → 記錄安全事件，302 重導向首頁；code 無效 → 302 重導向首頁 |

### 4.3 GET /api/auth/me
**用途**：取得當前登入使用者資訊

| 項目 | 內容 |
|------|------|
| 認證需求 | 不需要（未登入回傳 null） |
| 請求參數 | 無 |
| 成功回應 | `{ id, email, name, picture, isAdmin }` |
| 未登入回應 | `{ user: null }` |

### 4.4 POST /api/auth/logout
**用途**：登出並清除 Session

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要登入 |
| 請求參數 | 無 |
| 成功回應 | 302 重導向至首頁 |

## 5. Session Cookie 規格

| 屬性 | 值 |
|------|-----|
| Cookie 名稱 | `session` |
| 加密演算法 | AES-256-GCM |
| 金鑰來源 | 環境變數 `AUTH_SECRET`（需為 32 bytes hex，共 64 字元） |
| 格式 | `base64url(iv[12 bytes] + ciphertext + authTag[16 bytes])` |
| 有效期限 | 14 天 |
| httpOnly | true（防止 XSS 讀取） |
| SameSite | Lax |
| Secure | 生產環境自動啟用 |

**Session 內容（解密後）：**
```json
{
  "userId": "Google 使用者 ID",
  "email": "user@example.com",
  "name": "顯示名稱",
  "picture": "頭像 URL",
  "isAdmin": false
}
```

---

## 6. 安全機制

| 機制 | 說明 |
|------|------|
| PKCE (Proof Key for Code Exchange) | 防止授權碼攔截攻擊，每次登入產生隨機 `code_verifier` |
| State 參數驗證 | 防止 CSRF 攻擊，回調時比對 state 是否一致 |
| OAuth State 不符記錄 | 寫入 `security_events` 表，severity=high |
| httpOnly Cookie | Session 無法被 JavaScript 讀取 |
| AES-256-GCM 加密 | Session 內容完整性與機密性保護 |
| 管理員白名單 | 透過 `MOSCRIPT_ADMIN_EMAILS` 環境變數，不依賴資料庫 |

---

## 7. 相關環境變數

| 變數名稱 | 說明 | 必填 |
|---------|------|------|
| `GOOGLE_CLIENT_ID` | Google OAuth 應用程式 Client ID | 是 |
| `GOOGLE_CLIENT_SECRET` | Google OAuth 應用程式 Client Secret | 是 |
| `AUTH_SECRET` | Session 加密金鑰（openssl rand -hex 32） | 是 |
| `MOSCRIPT_APP_URL` | 應用程式完整 URL（用於 OAuth 回調） | 是 |
| `MOSCRIPT_ADMIN_EMAILS` | 管理員 email 清單（逗號分隔） | 否 |
