# F09 — 安全監控

> 功能類別：系統安全
> 相關檔案：`lib/security-log.ts`、`lib/rate-limit.ts`、`lib/validation.ts`、`next.config.ts`

---

## 1. 功能概述

系統內建多層安全機制，包含速率限制、安全事件記錄、輸入驗證、HTTP 安全標頭、以及 OAuth CSRF 防護。所有安全事件均寫入 `security_events` 資料表，管理員可透過後台 API 查詢監控。

---

## 2. 安全機制總覽

| 機制 | 說明 |
|------|------|
| 速率限制 | 限制 API 呼叫頻率，防止濫用 |
| OAuth CSRF 防護 | state + PKCE 雙重防護 |
| AES-256-GCM Session 加密 | Session 無法偽造或篡改 |
| HTTP 安全標頭 | CSP、HSTS、X-Frame-Options 等 |
| 輸入驗證 | 字段長度限制、中文字驗證 |
| 安全事件記錄 | 所有異常操作寫入日誌 |
| 私密資源存取控制 | 私密字圖需驗證所有者身份 |

---

## 3. 速率限制

### 3.1 限制規則

| 端點 | 限制 | Key |
|------|------|-----|
| `POST /api/glyphs/upload` | 10 次/分鐘 | IP 位址 |
| `POST /api/glyphs/[id]/like` | 30 次/分鐘 | 使用者 ID |

### 3.2 實作機制

- 使用 **記憶體儲存**（Map），伺服器重啟後重置
- 以滑動視窗（Sliding Window）計算請求次數
- 自動清理過期條目（每分鐘）
- 觸發後記錄 `security_events`（severity=medium）

### 3.3 觸發回應

```json
HTTP 429 Too Many Requests

{
  "error": "Rate limit exceeded. Please try again later."
}
```

## 4. 安全事件類型

| event_type | 說明 | severity |
|-----------|------|---------|
| `oauth_state_mismatch` | OAuth state 參數不符（疑似 CSRF） | high |
| `api_unauthorized` | API 需要登入但未登入 | low |
| `api_forbidden` | API 需要管理員但無權限 | medium |
| `rate_limit_upload` | 字圖上傳超過速率限制 | medium |
| `rate_limit_like` | 按讚超過速率限制 | medium |
| `resource_access_violation` | 嘗試存取無權限的私密資源 | high |

### 4.1 安全事件記錄格式

每筆記錄包含：

| 欄位 | 說明 |
|------|------|
| event_type | 事件類型代碼 |
| severity | 嚴重程度（high/medium/low） |
| ip | 來源 IP 位址 |
| user_agent | 瀏覽器/客戶端資訊 |
| user_id | 當前使用者 ID（若已登入） |
| path | 請求路徑 |
| details | 額外資訊（JSON 格式） |

## 5. HTTP 安全標頭

在 `next.config.ts` 中統一設定所有安全標頭：

| 標頭名稱 | 值 | 說明 |
|---------|-----|------|
| `X-Content-Type-Options` | `nosniff` | 防止 MIME 嗅探 |
| `X-Frame-Options` | `SAMEORIGIN` | 防止 Clickjacking |
| `X-XSS-Protection` | `1; mode=block` | 舊版瀏覽器 XSS 過濾 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 控制 Referrer 資訊洩漏 |
| `Content-Security-Policy` | 見下方 | 防止 XSS、資源注入 |
| `Strict-Transport-Security` | `max-age=63072000` | 強制 HTTPS（生產環境） |
| `Permissions-Policy` | 限制感應器等 API | 減少攻擊面 |

**CSP 設定（Content Security Policy）：**
```
default-src 'self';
script-src 'self' 'unsafe-inline' 'unsafe-eval';
style-src 'self' 'unsafe-inline';
img-src 'self' data: blob: https:;
font-src 'self';
connect-src 'self';
frame-ancestors 'none';
```

## 6. 輸入驗證

由 `lib/validation.ts` 提供統一的驗證與截斷函式：

| 驗證類型 | 說明 |
|---------|------|
| 中文字驗證 | 確保 `char` 欄位為有效 CJK 字元 |
| 字串長度截斷 | author ≤ 100、work_title ≤ 200、script_type ≤ 50 |
| 數字範圍驗證 | quality_score（整數）、perChar（1~10）|
| 圖片格式驗證 | MIME type 白名單驗證 |
| 檔案大小驗證 | 最大 20 MB |

## 7. 存取控制

### 7.1 私密字圖存取

由 `lib/glyph-access.ts` 的 `canAccessGlyph()` 函式判斷：

```
字圖存取判斷邏輯：
  ├── 字圖 visibility = 'public' → 允許任何人
  ├── 字圖 visibility = 'private'
  │     ├── 當前使用者 = 字圖 owner_user_id → 允許
  │     ├── 當前使用者 isAdmin = true → 允許
  │     └── 其他 → 拒絕（403 + 記錄安全事件）
  └── 字圖不存在 → 404
```

### 7.2 管理員功能存取

- Middleware 層（`middleware.ts`）：`/admin/*` 路由需通過 Session 驗證與 isAdmin 檢查
- API 層：各後台 API 重複驗證管理員身份（防止 Middleware 繞過）

## 8. OWASP Top 10 對應

| 風險 | 因應措施 |
|------|---------|
| A01 權限控制失效 | 存取控制函式 + Middleware 保護 |
| A02 加密機制失效 | AES-256-GCM Session、HTTPS HSTS |
| A03 注入攻擊 | Prepared Statement（better-sqlite3）、輸入驗證 |
| A05 安全性錯誤設定 | 安全標頭、CSP、移除伺服器版本資訊 |
| A06 元件漏洞 | 定期更新依賴（postcss 等） |
| A07 認證失效 | Google OAuth PKCE + state、加密 Session |
| A09 安全記錄不足 | 完整的 security_events 日誌 |
