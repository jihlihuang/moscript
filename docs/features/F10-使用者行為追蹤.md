# F10 — 使用者行為追蹤

> 功能類別：數據分析
> 相關檔案：`lib/usage-log.ts`、`app/api/glyphs/`

---

## 1. 功能概述

系統自動記錄使用者的主要行為事件（搜尋、上傳等）至 `usage_events` 資料表，供管理員分析使用模式、熱門搜尋詞彙、上傳成功率等指標，以利系統優化與書庫擴充決策。

---

## 2. 記錄的事件類型

| event_type | 觸發時機 | 主要資料 |
|-----------|---------|---------|
| `search` | 使用者執行字圖搜尋 | 搜尋詞、結果數、查詢時間(ms) |
| `upload_succeeded` | 字圖上傳成功 | 字、書體、可見性 |
| `upload_failed` | 字圖上傳失敗 | 失敗原因、字（若有） |

---

## 3. 事件資料格式

### 3.1 search 事件

```json
{
  "event_type": "search",
  "subject": "山水",
  "details": {
    "q": "山水",
    "scriptTypes": ["行", "草"],
    "resultScope": "library",
    "sort": "popular",
    "resultCount": 6,
    "durationMs": 12
  },
  "user_id": "google_user_id 或 null（訪客）"
}
```

**欄位說明：**

| 欄位 | 說明 |
|------|------|
| subject | 搜尋文字（方便直接 GROUP BY 統計） |
| q | 完整搜尋詞 |
| scriptTypes | 選取的書體陣列 |
| resultScope | 搜尋範圍 |
| sort | 排序方式 |
| resultCount | 回傳的字圖組數 |
| durationMs | SQL 查詢耗時（毫秒） |

### 3.2 upload_succeeded 事件

```json
{
  "event_type": "upload_succeeded",
  "subject": "山",
  "details": {
    "char": "山",
    "scriptType": "行",
    "visibility": "public"
  },
  "user_id": "google_user_id"
}
```

### 3.3 upload_failed 事件

```json
{
  "event_type": "upload_failed",
  "subject": "山",
  "details": {
    "char": "山（若已解析）",
    "reason": "檔案格式不支援"
  },
  "user_id": "google_user_id"
}
```

---

## 4. 隱私考量

| 項目 | 做法 |
|------|------|
| 訪客搜尋 | user_id 記錄為 null，不追蹤個人 |
| 不記錄個人識別資料 | 僅記錄行為類型與摘要 |
| 搜尋詞 | 記錄搜尋的字，不記錄 IP 或其他個人資料 |

---

## 5. 管理員查詢應用

透過 `GET /api/admin/stats`，管理員可基於 usage_events 取得：

| 統計項目 | SQL 概念 |
|---------|---------|
| 熱門搜尋字 | `SELECT subject, COUNT(*) FROM usage_events WHERE event_type='search' GROUP BY subject ORDER BY COUNT(*) DESC` |
| 每日搜尋量 | `GROUP BY DATE(created_at)` |
| 上傳成功率 | 比較 upload_succeeded vs upload_failed 數量 |
| 最常上傳的字 | `SELECT subject, COUNT(*) FROM usage_events WHERE event_type='upload_succeeded' GROUP BY subject` |
