# F08 — 後台管理

> 功能類別：系統管理
> 相關檔案：`app/admin/`、`app/api/admin/`、`app/api/glyph-sets/`、`components/AdminGlyphUploadForm.tsx`

---

## 1. 功能概述

後台管理功能專供管理員使用，提供字圖資料庫管理、字圖批次上傳、字組管理、系統統計、安全事件監控等功能。管理員身份由環境變數 `MOSCRIPT_ADMIN_EMAILS` 白名單控制。

---

## 2. 存取控制

| 保護機制 | 說明 |
|---------|------|
| Session 驗證 | 需要有效的加密 Session Cookie |
| 管理員驗證 | `isAdmin` 需為 true（email 在白名單中） |
| 未授權處理 | 未登入 → 重導向至首頁；非管理員 → 403 回應 |

---

## 3. 後台分頁功能

後台首頁（`/admin`）提供四個分頁：

### 3.1 系統狀態

| 統計項目 | 說明 |
|---------|------|
| 字圖總數 | 系統書庫字圖數量 |
| 書體分佈 | 各書體的字圖數量 |
| 熱門搜尋 | 最常搜尋的中文字 |
| 上傳成功率 | 近期上傳失敗統計 |

### 3.2 字圖管理

| 操作 | 說明 |
|------|------|
| 搜尋字圖 | 依字、作者、書體篩選 |
| 編輯字圖 | 修改元資料（PATCH /api/glyphs/[id]） |
| 刪除字圖 | 刪除字圖及實體檔案（DELETE /api/glyphs/[id]） |

### 3.3 字組管理

多字拆圖批次上傳後產生的字組在此集中管理：

| 功能 | 說明 |
|------|------|
| 列表瀏覽 | 固定高度卡片、字圖可捲動預覽 |
| 搜尋 | 依字組名稱、字圖、作者或作品搜尋（僅接受中文） |
| 編輯名稱 | 點擊管理可修改字組名稱 |
| 管理成員 | 展開後可調整字圖排序、新增/移除字圖 |
| 查看原圖 | 點擊縮圖可預覽拆字原稿 |
| 刪除字組 | 刪除字組（字圖資料保留，僅取消分組） |

### 3.4 資安監控

查看近期安全事件日誌（見 [F09-安全監控.md](F09-安全監控.md)）。

## 4. 後台字圖上傳（/admin/upload）

管理員專用的字圖上傳頁面，使用 `AdminGlyphUploadForm` 元件，支援單張與多字拆圖兩種模式。

### 4.1 單張上傳

逐一上傳，手動填寫每個字圖的元資料。前端支援：
- 圖片黑白化、對比調整
- 畫布擦除（BW 模式用白色填充，灰階模式用內容感知填充）
- 擦除歷史 Undo

### 4.2 多字拆圖批次上傳

從一張原稿圖自動拆分並批次上傳：

| 拆字方向 | 說明 |
|---------|------|
| 自動 | 系統判斷橫/直排 |
| 橫排 | 由左至右排列 |
| 直排 | 由上至下排列 |
| 手動框選 | 使用者自行框選每個字 |

上傳流程：

```
選擇原稿圖 → 選擇拆字方向 → 自動拆字
  │
  ├── 可逐一調整每字：
  │     ├── 修改裁切範圍
  │     ├── 擦除雜訊（灰階模式支援材質感填充）
  │     └── 切換顏色模式（黑白/灰階）
  │
  ├── 填入文字（支援整段文字快速填入）
  │
  └── 批次上傳
        ├── 建立字組（POST /api/glyph-sets）
        │     └── 名稱 = 批次字元合集，上傳原稿圖
        ├── 逐一上傳字圖（含 setId + setPosition）
        ├── 顯示進度條
        └── 全部成功後自動清除拆字結果
```

**必填欄位**（所有上傳模式共通）：
- 書體（scriptType）
- 作者
- 作品名稱
- 來源
- 授權類型

## 5. API 規格

### GET /api/admin/stats

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要管理員 |

回傳字圖總數、書體分佈、熱門搜尋字、上傳失敗統計等。

### POST /api/admin/upload

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要管理員 |
| Content-Type | `multipart/form-data` |

欄位同 `/api/glyphs/upload`，額外支援：`qualityScore`、`setId`、`setPosition`。

### GET /api/glyph-sets

| 項目 | 內容 |
|------|------|
| 認證需求 | 公開（管理員可查看全部，含無名稱字組） |

**Query 參數：**

| 參數 | 說明 |
|------|------|
| `q` | 搜尋關鍵字（名稱、字圖、作者、作品） |
| `adminMode` | `1` 時回傳包含無名稱的所有字組 |
| `limit` | 回傳筆數上限 |

### PATCH /api/glyph-sets/[id]

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要管理員或字組擁有者 |

可修改：`name`、字圖排序（`glyphIds` 陣列）。

### DELETE /api/glyph-sets/[id]

| 項目 | 內容 |
|------|------|
| 認證需求 | 需要管理員或字組擁有者 |
| 副作用 | 刪除字組記錄；字圖的 `set_id` 設為 NULL（字圖本身保留） |

### PATCH /api/glyphs/[id]（管理員編輯字圖）

可編輯所有字圖欄位：`char`、`author`、`script_type`、`work_title`、`source`、`license`、`quality_score`、`visibility`。

### DELETE /api/glyphs/[id]（管理員刪除字圖）

刪除字圖記錄及實體檔案，串聯刪除 `glyph_likes` 與 `collection_items`。

## 6. 管理員操作日誌

所有後台操作均記錄至 `admin_audit_logs` 表：

| action 值 | 操作說明 |
|-----------|---------|
| `glyph_upload` | 上傳字圖（含批次） |
| `glyph_update` | 更新字圖元資料 |
| `glyph_delete` | 刪除字圖 |
| `glyph_set_update` | 更新字組名稱或排序 |
| `glyph_set_delete` | 刪除字組 |
