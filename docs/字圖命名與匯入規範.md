# 字圖命名與匯入規範

> 適用對象：負責新增、整理書法字圖的內容維護人員
> 相關指令：`npm run import:glyphs`
> 相關腳本：`scripts/import-glyphs.ts`

---

## 1. 目錄結構規範

所有**系統書庫字圖**（公開、非個人上傳）必須放在：

```
public/
  glyphs/
    [中文字]/
      [作者]_[書體]_[作品名]_[編號].[副檔名]
```

每個中文字一個子目錄，目錄名稱即為該字本身（Unicode，如 `山`、`水`）。

---

## 2. 檔案命名規範

```
[作者]_[書體]_[作品名]_[編號].[副檔名]
```

**各欄位說明：**

| 欄位 | 說明 | 允許為空 | 範例 |
|------|------|---------|------|
| 作者 | 書法家姓名 | 是 | `王羲之`、`顏真卿` |
| 書體 | 書體類型 | 是 | `草`、`行`、`隸`、`楷`、`篆` |
| 作品名 | 作品標題 | 是 | `蘭亭集序`、`多寶塔碑` |
| 編號 | 同字同作品的序號 | 是 | `001`、`002` |

欄位之間以底線 `_` 分隔。若欄位為空，底線仍保留（但腳本會自動忽略前後多餘的底線）。

**完整範例：**
```
王羲之_行_蘭亭集序_001.jpg
顏真卿_楷_多寶塔碑_002.png
孫過庭_草_書譜_001.jpg
_楷__001.jpg             # 作者、作品名均不明
歐陽詢_楷_九成宮醴泉銘_003.webp
```

---

## 3. 支援的圖片格式

| 副檔名 | 說明 |
|--------|------|
| `.jpg` / `.jpeg` | 最常用，壓縮率高 |
| `.png` | 支援透明背景（適合白底字圖） |
| `.webp` | 現代格式，壓縮率優於 jpg |
| `.gif` | 支援（但通常不適合字圖） |
| `.svg` | 向量格式（適合描繪的字圖）；**匯入腳本支援，但瀏覽器 img 標籤直接顯示** |

> 建議統一使用 `.jpg`（高品質）或 `.png`（需透明背景時）。

---

## 4. 目錄範例

```
public/glyphs/
  山/
    王羲之_行_蘭亭集序_001.jpg
    顏真卿_楷_多寶塔碑_001.jpg
    孫過庭_草_書譜_001.jpg
    歐陽詢_楷_九成宮醴泉銘_001.jpg
  水/
    王羲之_行_蘭亭集序_001.jpg
    米芾_行__001.jpg
  風/
    _草__001.jpg
```

## 5. 匯入腳本執行方式

準備好圖片後，執行：

```bash
npm run import:glyphs
```

腳本輸出範例：
```
掃描 150 個圖片檔，新增 48 筆，目前資料庫共 2048 筆。
```

**腳本行為說明：**

| 行為 | 說明 |
|------|------|
| `INSERT OR IGNORE` | 若 `image_url` 已存在則跳過，不會重複匯入 |
| source | 自動設為 `local-dataset` |
| license | 自動設為 `non-commercial-research` |
| quality_score | 自動設為 `0`（需進後台手動調整） |
| owner_user_id | 設為 `NULL`（系統書庫字圖） |
| visibility | 預設 `public` |

---

## 6. 匯入後的後續操作

匯入後，建議透過後台管理介面調整：

### 6.1 調整品質分數（quality_score）

品質分數 0~10，影響 `popular` 排序中的預設排名。
- 精選字圖（清晰、代表性強）：建議設 7~10
- 一般字圖：建議設 3~6
- 品質較差的字圖：建議設 1~2 或考慮不匯入

### 6.2 確認書體填寫

書體欄位影響篩選功能，確保填寫正確（草、行、隸、楷、篆、魏、其他）。

### 6.3 確認作者名稱一致性

同一書法家需確保命名一致（如「王羲之」不可混用「王右軍」），否則按作者篩選時會分裂。

---

## 7. 進階：圖片品質建議

| 項目 | 建議 |
|------|------|
| 解析度 | 最短邊 300px 以上；列表縮圖用 150×150，詳情頁用原圖 |
| 背景 | 白底或透明背景（避免複雜背景干擾辨識） |
| 裁切 | 字圖應佔畫面 70% 以上，避免過多留白 |
| 色彩 | 黑白或深色墨跡為主，避免彩色底紙 |
| 壓縮 | jpg 建議 quality 85~95；不建議過度壓縮 |

---

## 8. 個人上傳 vs 系統書庫

| 項目 | 系統書庫字圖 | 個人上傳字圖 |
|------|-----------|-----------|
| 存放路徑 | `public/glyphs/` | `public/glyphs/`（公開）或 `data/private-glyphs/`（私密） |
| 匯入方式 | `npm run import:glyphs` 或後台上傳 | 使用者透過 `/upload` 頁面 |
| `owner_user_id` | NULL | 使用者 Google ID |
| `source` | `local-dataset` 或 `import` | `personal-upload` |
| 可被所有人搜尋 | 是（resultScope=library） | 需設為 public，且 resultScope=all 或 public |

---

## 9. 常見錯誤

### 檔案已存在（`image_url` 衝突）

腳本使用 `INSERT OR IGNORE`，已存在的 `image_url` 不會重新插入。若需更新元資料，請透過後台編輯介面或直接修改資料庫。

### 字符目錄名稱不是中文字

腳本不驗證目錄名稱是否為中文字，但搜尋功能只能查詢 CJK 字元，非中文目錄的字圖不會出現在搜尋結果中。

### 路徑含特殊字元

中文路徑會以 `encodeURIComponent` 編碼後儲存為 `image_url`，瀏覽器讀取時自動解碼，無需擔心。
