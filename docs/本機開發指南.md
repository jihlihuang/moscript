## 2. 初次設定

### 2.1 安裝依賴

```bash
npm install
```

### 2.2 設定環境變數

複製範例檔並填入實際值：

```bash
cp .env.deploy.example .env.local
```

本機開發最低限度的 `.env.local` 設定（Google OAuth 才能登入）：

```bash
# Google OAuth（需至 Google Cloud Console 建立 OAuth 應用）
GOOGLE_CLIENT_ID=你的_client_id
GOOGLE_CLIENT_SECRET=你的_client_secret

# Session 加密金鑰（執行下方指令產生）
AUTH_SECRET=產生後填入

# 本機開發 URL
MOSCRIPT_APP_URL=http://localhost:3000

# 管理員帳號（填入自己的 Google 帳號 email）
MOSCRIPT_ADMIN_EMAILS=your@gmail.com
```

產生 `AUTH_SECRET`：

```bash
openssl rand -hex 32
```

**Google OAuth 設定步驟（首次）：**
1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立新專案或選擇現有專案
3. 啟用 Google+ API
4. 前往「憑證」→「建立憑證」→「OAuth 2.0 用戶端 ID」
5. 應用程式類型選「網頁應用程式」
6. 已授權的重新導向 URI 填入：`http://localhost:3000/api/auth/google/callback`
7. 複製 Client ID 與 Client Secret 填入 `.env.local`

### 2.3 啟動開發伺服器

```bash
npm run dev
```

開啟 `http://localhost:3000`。

## 3. 新增書法字圖（本機資料匯入）

### 3.1 準備圖片

將圖片依以下結構放入 `public/glyphs/`：

```
public/glyphs/
  山/
    王羲之_行_蘭亭集序_001.jpg
    顏真卿_楷_多寶塔碑_002.png
  水/
    孫過庭_草_書譜_001.jpg
```

**命名規則詳見：[字圖命名與匯入規範.md](字圖命名與匯入規範.md)**

### 3.2 執行匯入腳本

```bash
npm run import:glyphs
```

腳本會掃描 `public/glyphs/` 所有圖片，自動寫入 `data/moscript.sqlite`，跳過已存在的記錄（`INSERT OR IGNORE`）。

### 3.3 確認匯入結果

```bash
sqlite3 data/moscript.sqlite "SELECT COUNT(*) FROM glyphs;"
sqlite3 data/moscript.sqlite "SELECT char, author, script_type FROM glyphs LIMIT 10;"
```

## 4. 常用指令

| 指令 | 說明 |
|------|------|
| `npm run dev` | 啟動開發伺服器（http://localhost:3000） |
| `npm run build` | 建置正式版本 |
| `npm run start` | 啟動正式版本（需先 build） |
| `npm run import:glyphs` | 掃描 public/glyphs/ 並匯入資料庫 |
| `npm run upload:blob` | 上傳資料庫與字圖至 Azure Blob Storage |
| `npm run deploy:azure` | 部署至 Azure Container Apps |
| `npm run restart:azure` | 重啟 Azure Container App |

## 5. 專案結構說明

```
moscript/
├── app/                    # Next.js App Router（頁面與 API）
│   ├── api/                # API 路由（Server-side）
│   ├── admin/              # 後台頁面（需管理員）
│   ├── me/                 # 個人頁面
│   ├── collections/        # 集字作品
│   ├── glyph/[id]/         # 字圖詳情
│   ├── practice/[id]/      # 書法練習
│   └── upload/             # 上傳字圖
├── components/             # 可重用 React 元件
├── lib/                    # 核心邏輯（資料庫、認證、搜尋等）
├── scripts/                # 維運腳本（匯入、部署、Blob 上傳）
├── public/glyphs/          # 公開字圖（靜態檔案）
├── data/                   # 本機資料（SQLite 與私密字圖）
│   ├── moscript.sqlite     # 主資料庫
│   └── private-glyphs/     # 私密字圖（不公開）
├── docs/                   # 技術文件（本文件所在位置）
└── .env.local              # 本機環境變數（不進版控）
```

## 6. 後台管理員

登入後，若你的 email 在 `MOSCRIPT_ADMIN_EMAILS` 清單中，即可進入後台：

```
http://localhost:3000/admin
```

後台功能：
- 查看資料庫統計
- 上傳字圖至系統書庫
- 查看安全事件日誌

## 7. 資料庫直接操作

SQLite 資料庫位於 `data/moscript.sqlite`（本機開發）。

**常用查詢：**

```bash
# 查看所有資料表
sqlite3 data/moscript.sqlite ".tables"

# 查看字圖數量（依書體分組）
sqlite3 data/moscript.sqlite "SELECT script_type, COUNT(*) FROM glyphs GROUP BY script_type;"

# 查看最新上傳的 10 筆
sqlite3 data/moscript.sqlite "SELECT id, char, author, created_at FROM glyphs ORDER BY id DESC LIMIT 10;"

# 刪除測試資料
sqlite3 data/moscript.sqlite "DELETE FROM glyphs WHERE source='test';"
```

## 8. 常見問題

### Q: 登入後顯示「未授權」

確認 `MOSCRIPT_ADMIN_EMAILS` 包含你的 Google 帳號 email，且格式正確（逗號分隔，無空格）。

### Q: 搜尋不到任何字圖

執行 `npm run import:glyphs` 匯入字圖。確認 `data/moscript.sqlite` 存在：
```bash
sqlite3 data/moscript.sqlite "SELECT COUNT(*) FROM glyphs;"
```

### Q: Google OAuth 登入失敗（redirect_uri_mismatch）

確認 Google Cloud Console 的「已授權的重新導向 URI」包含 `http://localhost:3000/api/auth/google/callback`。

### Q: 上傳圖片後看不到

- 公開圖片：確認圖片在 `public/glyphs/` 並已執行匯入
- 私密圖片：確認已登入，且圖片 URL 走 `/api/glyphs/[id]/asset` 路徑

### Q: 如何重置資料庫

```bash
rm data/moscript.sqlite
npm run import:glyphs  # 重新匯入
```
